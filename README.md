# SolarkBot Photo Atelier

A cinematic Next.js experience for `nft.solarkbot.xyz` where the SolarkBot artist turns a prompt into artwork, reveals it fullscreen, lets the user download it, and optionally mints it as an NFT on Solana testnet.

## Stack

- Next.js 16 App Router with TypeScript
- Tailwind CSS v4
- React Three Fiber + drei
- Framer Motion
- Zustand
- NVIDIA hosted image generation API
- Solana wallet adapter + Metaplex UMI
- Pinata IPFS uploads
- Vitest + Testing Library + Playwright

## Local Setup

1. Install dependencies:

```bash
npm install
```

2. Copy `.env.example` to `.env.local` and fill in the required secrets.

3. Start the dev server:

```bash
npm run dev
```

4. Open [http://localhost:3000](http://localhost:3000).

## Environment

Required server variables:

- `NVIDIA_API_KEY`

- `PINATA_GATEWAY_URL`

- `NEXT_PUBLIC_SOLANA_NETWORK`
- `NEXT_PUBLIC_SOLANA_RPC_URL`

## How Image Generation Works

1. `POST /api/generate-image` validates the prompt, negative prompt, and aspect ratio.
2. The route calls `lib/nvidia/generateImage.ts`, which wraps the NVIDIA API and maps supported aspect ratios to the official SD 3.5 Large resolutions.
3. The generated bytes are converted to an inline `data:image/...;base64,...` URL on the server.
4. The client receives a frontend-safe payload containing `imageUrl`, `artifactId`, dimensions, and prompt echoes.
5. The reveal screen lets the user download the finished image, mint it on testnet, or return to the studio to create another.

If NVIDIA changes the hosted endpoint path or request schema, update `lib/nvidia/generateImage.ts`. That file remains the single integration boundary for the image provider.

## Download Flow

- The reveal stage works directly from the returned inline image data.
- The client saves it with the format `solarkbot-art-YYYYMMDD-HHmmss.png`.

## Minting Flow

1. The reveal stage exposes `Mint as NFT` after generation.
2. `POST /api/mint-nft` uploads the generated image to Pinata and then uploads NFT metadata JSON to Pinata.
3. The client connects **Phantom** on `testnet`, creates a standalone NFT with Metaplex UMI, and mints it to the connected wallet.
4. After mint success, the reveal shows a Solscan testnet link for the token.

Desktop minting expects the Phantom extension. On mobile, the first pass hands the user into Phantom to finish the mint there.

Pinata credentials remain server-only. If Pinata changes its upload API or auth expectations, update `lib/ipfs/pinata.ts`.

## Solana Network

- The app currently targets `testnet`.
- Switch `SOLANA_NETWORK`, `NEXT_PUBLIC_SOLANA_NETWORK`, and `NEXT_PUBLIC_SOLANA_RPC_URL` together when you want to move networks.
- The current mint is a standalone NFT. It is not a verified collection mint and not a compressed NFT.

## Known Assumptions

- The artist is a stylized primitive stand-in rather than a rigged character, with camera choreography and prop motion carrying the performance.
- Generated images are returned inline and are not persisted between sessions.
- The current Solana mint flow expects Phantom and a wallet funded for the active cluster's transaction fees.
